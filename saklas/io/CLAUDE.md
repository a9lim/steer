# io/

Persistence + distribution: packs, HF hub, GGUF, cloning, datasource.

## packs.py

`PackMetadata` + `ConceptFolder`. `PACK_FORMAT_VERSION = 2`; loader raises `PackFormatError` on explicit v1 with a `scripts/upgrade_packs.py` hint. `materialize_bundled` detects stale v1 on bundled concepts and upgrades in place.

Shared helpers used by both `hf.py::_install_synthesized_pack` (pack-less HF repos) and `session.py::_update_local_pack_files`:
- `synthesize_pack_metadata(*, name, source, pack_dir, description="", tags=(), version="1.0.0", license="unknown", recommended_alpha=0.5, long_description="")`
- `hash_folder_files(folder)`

Divergent fields (version, license, description, source) are helper params, so each caller preserves its existing pack.json bytes on disk.

## hf.py

HF distribution. Packs are **model repos** (not datasets) — safetensors is model-hub-native and `base_model` frontmatter gives reverse-link discoverability. `repo_type="model"` exclusively. `saklas install owner/name@rev` pins to any git ref (tag/branch/SHA); `split_revision` threads `revision` through download/refresh.

**Pack-less install**: `install owner/name[@rev]` synthesizes `pack.json` when the HF repo has no root manifest — scans `*.safetensors`/`*.gguf`, fabricates `method="imported"` sidecars for bare safetensors, computes hashes, slugs `name` from the repo name via `NAME_REGEX`, sets `source = "hf://..."`. Repeng-style GGUF-only repos (`jukofyork/creative-writing-control-vectors-v3.0`, etc.) install with zero prep. Subsequent ops treat the folder identically to native packs; `push` still requires `-f` + `-a` (same guard as pulled HF packs).

**`saklas pack push`**: publishes one concept folder as an HF model repo. Default coord `<whoami>/<pack_name>`; `--as/-a` overrides. `-p` private, `-m MODEL` scope tensors, `-s` statements-only, `-n` tensors-without-statements, `-t` tag `v<pack.version>`, `-d` dry-run (stages README/.gitattributes/filtered pack.json), `-f` force-through for non-owned packs. Model card YAML carries `library_name: saklas`, merged tags (`saklas-pack`, `activation-steering`, `steering-vector`, + `pack.tags`), a `base_model:` list from sidecar filenames, `base_model_relation: adapter`. `.gitattributes` pins `*.safetensors` to LFS.

`_download` upgrades the error message when a user points at a dataset repo instead of a model repo.

## gguf_io.py

`write_gguf_profile(profile, path, *, model_hint)` + `read_gguf_profile(path)`. Validates `controlvector` architecture field, requires `model_hint`, parses `direction.*` tensor names for layer indices, returns `(profile, {method: "gguf_import", ...})`. Lazy import of `gguf`; raises `GGUFNotInstalled` (ImportError subclass) with install hint.

**`saklas pack export gguf`**: writes baked tensors to llama.cpp's control-vector GGUF format. Wire: `general.architecture = "controlvector"`, `controlvector.model_hint = <arch>` (via `transformers.AutoConfig.model_type`; `--model-hint` overrides), `controlvector.layer_count`, tensors `direction.<layer_idx>` as fp32. Because shares are baked, llama.cpp's uniform `--control-vector-scaled` scalar reproduces saklas's per-layer weighting for free — no per-layer metadata needed.

**Bundled concepts refuse in-place export** (their folder is restored on `refresh`; any in-place GGUF would silently vanish) — pass `-o` pointing outside the pack folder.

Repeng unit-normed GGUFs round-trip fine; saklas-baked GGUFs carry share-weighted magnitudes.

## probes_bootstrap.py

Walks `~/.saklas/vectors/default/` after `materialize_bundled()`. 21 bundled probes (19 bipolar + 2 monopolar; see `regenerate_bundled_statements.py` manifest) categorized via `pack.json` tags (`affect`, `epistemic`, `alignment`, `register`, `social_stance`, `cultural`). `bootstrap_layer_means` caches `~/.saklas/models/<id>/layer_means.safetensors`, stale when `neutral_statements.json` hash changes. MPS cache flushed between probes.

## cloning.py

Training-free persona cloning. `clone_from_corpus(session, path, name, *, n_pairs=90, seed=None, batch_size=5, force=False)` reads a one-utterance-per-line text file, samples `n_pairs` exemplars, and pairs each against a *neutralized* rewrite generated by the same model in batches of `batch_size` (default `_BATCH_SIZE = 5` — larger models can take bigger batches, small/MoE may need smaller).

**Delegates to `session.extract(DataSource(pairs=...))`** for the contrastive PCA + save — cloning owns only corpus hashing, exemplar sampling, rewrite batching, and the cache short-circuit. Result saved to `local/<name>/`; `pack.json` overwritten with clone metadata (`corpus_sha256`, `n_pairs`, `batch_size`, `seed`, `tags += ["cloned"]`).

Cache key: `sha256(corpus) + n_pairs + batch_size + seed` compared against existing `pack.json`; `force=True` bypasses. `session.clone_from_corpus(...)` is the thin public entry point.

## SAE variant coexistence

A concept folder can hold multiple baked tensors per model, distinguished by filename suffix: `<safe_model>.safetensors` (raw PCA) and `<safe_model>_sae-<release>.safetensors` (SAE-PCA, one per release). `safe_variant_suffix(release)` slugs unsafe chars; `tensor_filename(model_id, release=...)` / `sidecar_filename(...)` / `parse_tensor_filename(name)` in `paths.py` are the canonical constructors and inverse. `enumerate_variants(folder, model_id)` in `packs.py` returns `{"raw" | "sae-<release>": Path}`.

Sidecar JSON gets three additional optional fields when `method == "pca_center_sae"`: `sae_release`, `sae_revision`, `sae_ids_by_layer`. Additive on top of `PACK_FORMAT_VERSION = 2` — no migration required. `pack.json.files` hashes every variant, so `ConceptFolder`-style integrity checks verify both raw and SAE tensors on every load.

`cache_ops.delete_tensors(selector, model_scope, *, variant="all")` and `hf.push_pack(..., variant="all")` both accept a variant filter: `"raw"` / `"sae"` / `"all"`. CLI defaults encode the UX intent — `pack clear --variant all`, `pack push --variant raw` (users opt into sharing SAE provenance explicitly).
