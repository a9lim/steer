# CLAUDE.md

## What this is

`saklas` is a Python library + Textual TUI + dual-protocol HTTP server (OpenAI `/v1/*` **and** Ollama `/api/*` on the same port) for activation steering and trait monitoring on HuggingFace causal LMs. Representation Engineering via contrastive PCA; per-call alpha control; no model mutation. Three frontends over one engine: `SaklasSession` (programmatic), `saklas serve` (HTTP), `saklas` (TUI).

Version `1.4.0`. Lives in **two places** ‚Äî bump both or `saklas.__version__` drifts from PyPI:
- `pyproject.toml` ‚Üí `[project] version`
- `saklas/__init__.py` ‚Üí `__version__`

Releases: merge bump to `main` ‚Üí `.github/workflows/release.yml` tags `v$VERSION`, builds, publishes via trusted publishing, cuts a GitHub release. Push without bumping ‚Üí no-op.

## Commands

```bash
pip install -e ".[dev]"                # editable + pytest
pip install -e ".[serve]"              # fastapi + uvicorn
pip install -e ".[gguf]"               # llama.cpp GGUF I/O
pip install -e ".[cuda]"               # bitsandbytes + flash-attn (Linux/CUDA)
saklas <model_id>                       # TUI
saklas serve <model_id>                 # OpenAI + Ollama API (dual-protocol)
saklas install <target> [-s|-a NS/N|-f] # HF coord or folder; -s = statements only
saklas refresh <selector> [-m MODEL]    # re-pull; `refresh neutrals` is reserved
saklas clear <selector> [-m MODEL]      # delete per-model tensors (keep statements)
saklas uninstall <selector> [-y]        # remove folder (bundled respawns)
saklas list [selector] [-i|-j|-v]       # installed + HF by default
saklas merge <name> <components> [-m]   # merge baked tensors
saklas push <selector> [-a OWNER/NAME] [-p] [-m MODEL] [-s|-n] [-t] [-d] [-f]
saklas export gguf <selector> [-m MODEL] [-o PATH] [--model-hint HINT]
saklas clone <corpus> -N NAME [-m MODEL] [-n N_PAIRS] [--seed S] [-f]
saklas extract <concept>|<pos> <neg> [-m MODEL] [-f]
pytest tests/                           # all; GPU tests gated on CUDA/MPS
```

`_SUBCOMMAND_TABLE` in `cli.py` maps `{serve, install, refresh, clear, uninstall, list, merge, push, export, clone, extract}` ‚Üí `(parser_builder, runner)` tuples. CLI peeks `argv[0]`; anything else is bare-TUI (`args.command == "tui"` branch). Bare TUI: `<model>` + `-q/-d/-p/-s/--max-tokens` + repeatable `-c/--config` + `--strict`.

Selector grammar (shared): `<name>`, `<ns>/<name>`, `tag:<t>`, `namespace:<ns>`, `default`, `all`. Bare names resolve cross-namespace and raise `AmbiguousSelectorError` on collision. `refresh` silently skips `source=local`; `refresh neutrals` rewrites `~/.saklas/neutral_statements.json` from the bundled copy. `uninstall` refuses broad selectors without `-y`; bundled concepts respawn on next session init.

## Cache layout

All state under `~/.saklas/` (override via `SAKLAS_HOME`):

```
~/.saklas/
  neutral_statements.json              # user-editable (copy-on-miss from package)
  vectors/
    default/<concept>/                 # bundled (source=bundled)
      pack.json                        # name, description, tags, files{sha256}
      statements.json                  # contrastive pairs
      <safe_model_id>.safetensors      # baked tensor (native)
      <safe_model_id>.json             # slim sidecar: method/saklas_version/statements_sha256
      <safe_model_id>.gguf             # optional llama.cpp parallel
    local/<concept>/                   # user-authored (source=local; `refresh` skips)
    <hf_owner>/<concept>/              # HF-pulled (source=hf://owner/name[@rev])
  models/<safe_model_id>/
    layer_means.safetensors            # probe-centering baseline
    layer_means.json                   # hash of neutral_statements.json
```

- `pack.json.files` sha256 map verified on every `ConceptFolder.load`.
- Slim sidecars ‚Äî shares are baked into tensor magnitudes at extraction, so no `scores` field anymore.
- **Dual-format tensors**: `ConceptFolder` globs `*.safetensors` + `*.gguf`. Safetensors wins on same-stem conflict (native, carries sidecar). GGUF metadata lives in the header, no external sidecar ‚Äî `sidecar(sid)` raises `KeyError` for GGUF-only entries. `vectors.load_profile` dispatches on extension; both paths return `(profile, metadata)`.
- **Upgrade gotcha**: `materialize_bundled` is copy-on-miss for `neutral_statements.json`. Releases changing neutrals need `saklas refresh neutrals`; layer means auto-recompute via hash check in `bootstrap_layer_means`.

**HF distribution.** Packs are **model repos** (not datasets) ‚Äî safetensors is model-hub-native and `base_model` frontmatter gives reverse-link discoverability. `saklas/hf.py` uses `repo_type="model"` exclusively. `saklas install owner/name@rev` pins to any git ref (tag/branch/SHA); `hf.split_revision` threads `revision` through download/refresh so pinning is pinning.

**Pack-less install.** `install owner/name[@rev]` synthesizes `pack.json` when the HF repo has no root manifest: scans `*.safetensors`/`*.gguf`, fabricates `method="imported"` sidecars for bare safetensors, computes hashes, slugs `name` from the repo name via `NAME_REGEX`, sets `source = "hf://..."`. Repeng-style GGUF-only repos (`jukofyork/creative-writing-control-vectors-v3.0`, etc.) install with zero prep. Subsequent ops treat the folder identically to native packs; `push` still requires `-f` + `-a` (same guard as pulled HF packs).

**`saklas push`.** Publishes one concept folder as an HF model repo. Default coord `<whoami>/<pack_name>`; `--as/-a` overrides. `-p` private, `-m MODEL` scope tensors, `-s` statements-only, `-n` tensors-without-statements, `-t` tag `v<pack.version>`, `-d` dry-run (stages README/.gitattributes/filtered pack.json), `-f` force-through for non-owned packs. Model card YAML carries `library_name: saklas`, merged tags (`saklas-pack`, `activation-steering`, `steering-vector`, + `pack.tags`), a `base_model:` list from sidecar filenames, `base_model_relation: adapter`. `.gitattributes` pins `*.safetensors` to LFS.

**`saklas export gguf`.** Writes baked tensors to llama.cpp's control-vector GGUF format. Wire: `general.architecture = "controlvector"`, `controlvector.model_hint = <arch>` (via `transformers.AutoConfig.model_type`; `--model-hint` overrides), `controlvector.layer_count`, tensors `direction.<layer_idx>` as fp32. Because shares are baked, llama.cpp's uniform `--control-vector-scaled` scalar reproduces saklas's per-layer weighting for free ‚Äî no per-layer metadata needed. **Bundled concepts refuse in-place export** (their folder is restored on `refresh`; any in-place GGUF would silently vanish) ‚Äî pass `-o` pointing outside the pack folder.

## Architecture

Five layers: **model/vector**, **steering/monitoring**, **session**, **TUI**, **server**.

### Model + vector
- `model.py` ‚Äî HF causal LM loading. `_LAYER_ACCESSORS` maps `model_type` ‚Üí layer-list accessor (`def`, not lambdas). **Tested and known working** (= `_TESTED_ARCHS` frozenset): `qwen2`, `qwen3`, `gemma2`, `gemma3`, `mistral3`, `gpt_oss`, `llama`, `glm`. **Wired up but untested**: mistral/mixtral, phi/phi3/phimoe, cohere 1‚Äì2, deepseek v2/v3, starcoder2, olmo 1‚Äì3 + moe, granite/granitemoe, nemotron, stablelm, gpt2/neo/j/bigcode/neox, bloom, falcon/falcon_h1, mpt, dbrx, opt, recurrent_gemma. `get_layers()` emits a one-time `UserWarning` (gated by `_warned: set[str]`) when a loaded model's `model_type` isn't in `_TESTED_ARCHS` ‚Äî table still resolves, user just gets a flag. Be careful claiming breadth in user-facing docs ‚Äî the accessor table being 54 entries is not the same as 54 architectures working. Adding a new architecture = one entry. Cascading fallbacks on attention impl (SDPA ‚Üí eager), dtype, device. `_load_text_from_multimodal` extracts text-only sub-models (Ministral-as-Mistral3), strips `language_model.` prefixes, dequantizes FP8 inline. Patches `torch.histc` for MPS MoE routing.
- `vectors.py` ‚Äî One forward pass per prompt; `_capture_all_hidden_states` hooks every layer in a single pass. `_encode_and_capture_all` pools from the **last content token**, walking back past `tokenizer.all_special_ids` to skip trailing chat-template markers (Llama `<|eot_id|>`, Gemma `<end_of_turn>`, Qwen `<|im_end|>`). Contrastive diffs cast to **fp32** before differencing (fp16 loses precision on close vectors, makes SVD degenerate). Multi-pair: batched SVD across all layers (one `torch.linalg.svd` call on stacked `(n_layers, N, dim)`), scored by explained-variance ratio. Single-pair: scored by `diff_norm / activation_norm`. **Shares baked at extraction**: each direction scaled to mean activation norm of its layer, then multiplied by `score_i / sum(scores)`; hook-side math collapses to flat `user_alpha * _STEER_GAIN * sum(baked)` (all-zero fallback to uniform). Returns a **profile**: `dict[int, Tensor]` covering every layer ‚Äî no scores dict. `compute_layer_means` averages 90 neutral prompts for centering. `load_profile` dispatches on extension. `_template_overhead_cache` is keyed by `id(tokenizer)` ‚Äî safe only while one tokenizer lives the session lifetime.
- `gguf_io.py` ‚Äî `write_gguf_profile(profile, path, *, model_hint)` + `read_gguf_profile(path)`. Validates `controlvector` architecture field, requires `model_hint`, parses `direction.*` tensor names for layer indices, returns `(profile, {method: "gguf_import", ...})`. Lazy import of `gguf`; raises `GGUFNotInstalled` (ImportError subclass) with install hint. Repeng unit-normed GGUFs round-trip fine; saklas-baked GGUFs carry share-weighted magnitudes.
- `probes_bootstrap.py` ‚Äî Walks `~/.saklas/vectors/default/` after `materialize_bundled()`. 21 bundled probes (19 bipolar + 2 monopolar) categorized via `pack.json` tags (`affect`, `epistemic`, `alignment`, `register`, `social_stance`, `cultural`). `bootstrap_layer_means` caches `~/.saklas/models/<id>/layer_means.safetensors`, stale when `neutral_statements.json` hash changes. MPS cache flushed between probes.

### Steering + monitoring
- `hooks.py` ‚Äî `SteeringHook` adds a pre-composed vector to hidden states and **unconditionally rescales each position back to its pre-injection norm** (in-place, `torch.linalg.vector_norm(dtype=float32)` to avoid fp16 sum-of-squares overflow at hidden_dim ‚â• 2048). No flag, no escape hatch. Prevents crank-alpha-gibberish by keeping the residual stream on its natural magnitude trajectory while still rotating toward the pole. `SteeringManager` groups vectors by layer, sums co-layer directions into **one pre-composed hook per active layer**. `add_vector(name, profile, alpha)`. **Flat-scalar apply**: `effective_injection = user_alpha * _STEER_GAIN * sum(baked)` ‚Äî no per-layer score lookup, no `sum(scores)` division. Preserves **layer-count invariance** (different depths hit same behavior at same Œ±) and **score-magnitude invariance** (absolute PCA scores don't matter, only relative shares) ‚Äî both just moved from apply-time to extract-time. `_STEER_GAIN = 2.0` calibrated on gemma-4-31b-it with the 21-probe pack; v1.3.1 Œ±-sweep on `angry.calm` post-norm-preservation: coherent band Œ±‚âà0.4‚Äì0.8, cliff Œ±‚âà1.0. Smaller/MoE/heavily-safety-trained models may still need proportionally higher Œ± for residual architectural effects this doesn't capture. `HiddenCapture` is the companion class used by session + TUI: `attach(layers, layer_indices)` / `detach()` / `stacked()` ‚Äî each capture is `detach().clone()` of `output[0, -1, :]` (device-local, no sync); k-th capture = "state that produced generated token k".
- `monitor.py` ‚Äî `TraitMonitor` scores probes against per-layer hidden states via `_score_probes` (shared `_normalize_hidden` helper mean-centers with layer means, L2-normalizes, magnitude-weighted cosine sim). **Per-layer weight = `||baked||`** (= `share_L * ref_norm_L`) ‚Äî recovers the "how much does this layer read per unit Œ±" quantity from the tensor magnitude itself. `_ensure_cache` builds a **per-layer stacked cache** `{layer_idx: (V[P,D], W[P])}` where `V` holds unit-normed probe directions (zero rows for probes missing that layer) and `W[p]` = `||baked_p_L||` (0 when missing). One matmul per layer scores every probe against a hidden state in a single kernel launch; a single `.cpu().tolist()` at the end pulls the final aggregate out ‚Äî the scoring hot path has **zero `.item()` calls** regardless of probe count. Entry points: `score_per_token(captured, generated_ids, tokenizer, *, accumulate)` is primary ‚Äî takes per-layer `(n_tokens, dim)` captures, returns `(aggregate_vals, per_token_scores)`, updates history when `accumulate=True`. Aggregate pools from the last non-special token. `measure_from_hidden(hidden_per_layer)` is the test-path pre-aggregated entry. `measure(model, tokenizer, layers, text)` still runs a forward pass ‚Äî kept as a convenience for scoring arbitrary text outside a generation run. `_pending_aggregate` / `_pending_per_token` split so the TUI poll sees aggregate readiness independently. `add_probe`/`remove_probe` invalidate the stacked cache wholesale (rebuilt lazily next score). `probe_names`, `profiles`, `layer_means` are properties (no defensive copies).

### Session
- `session.py` ‚Äî `SaklasSession` owns model, profile registry (`_profiles`), monitor, `SteeringManager`, `HiddenCapture`, generation config, conversation history. **Vectors register without alphas** via `steer(name, profile)`; alphas are per-generation via `generate(input, alphas={"name": 0.5})`. Thinking is per-call, gated by `supports_thinking`; session decodes only the response portion into history. **No persistent steering hooks.** **Monitor scoring is in-flight, not post-hoc**: `_begin_capture()` attaches `HiddenCapture` on the union of probe layers before `generate_steered`; `_end_capture()` detaches in the finally. `_finalize_generation` ‚Üí `score_captured(generated_ids, accumulate=not stateless)` ‚Üí `monitor.score_per_token`. No second forward pass. **`_gen_active: bool` re-entry guard** set at the entry of `generate` / `generate_stream` (try/finally pair) ‚Äî public methods raise `RuntimeError` if re-entered; prevents hook leaks from pending-action double-dispatch. Guard lives at public entry points only, not in `_begin_capture` / `_apply_steering`, so a normal `_apply_steering` ‚Üí `_begin_capture` pairing inside a single gen doesn't self-trip. **Regime note**: scoring uses actual generation-regime hidden states (with full prompt context); `layer_means` are still computed wrap-assistant ‚Äî mean-centering is slightly off-regime but neutral drift is small. Recompute via `refresh neutrals` if you see bias. `_last_per_token_scores` stashed for TUI. OpenAI-parity sampling: `stateless`, `seed`, `stop`, `logit_bias`, `presence_penalty`, `frequency_penalty`, `logprobs`. Server always `stateless=True`; TUI uses stateful path. `_generation_preamble()` dedupes blocking+streaming setup. Streaming worker emits `None` as done sentinel in `finally`. `on_token(text, thinking, token_id, logprob, top_logprobs)`. Full custom-concept pipeline: cache ‚Üí curated dataset ‚Üí statement cache ‚Üí model-generated pairs ‚Üí contrastive PCA ‚Üí save. Curated concepts save under `default/<c>/`; user concepts under `local/<c>/`. Statement caches are model-independent (stored alongside tensor folder as `statements.json`). `_update_local_pack_files` refreshes `pack.json.files` via the shared `hash_folder_files` helper in `packs.py`. `_N_PAIRS = 45`. `canonical_concept_name(concept, baseline=None)` slugs poles via `[^a-z0-9]+ ‚Üí _` and joins bipolar via `BIPOLAR_SEP = "."`; `extract()` returns `(canonical_name, profile)`. Extract lookup scans `cli_selectors._all_concepts()` across every namespace, so `/steer bob/deer.wolf` hits HF-pulled packs without being bundled.
- `cloning.py` ‚Äî Training-free persona cloning. `clone_from_corpus(session, path, name, *, n_pairs=90, seed=None, batch_size=5, force=False)` reads a one-utterance-per-line text file, samples `n_pairs` exemplars, and pairs each against a *neutralized* rewrite generated by the same model in batches of `batch_size` (default `_BATCH_SIZE = 5` ‚Äî larger models can take bigger batches, small/MoE may need smaller). **Delegates to `session.extract(DataSource(pairs=...))`** for the contrastive PCA + save ‚Äî cloning owns only corpus hashing, exemplar sampling, rewrite batching, and the cache short-circuit. Result is saved to `local/<name>/` and then `pack.json` is overwritten with clone-specific metadata (`corpus_sha256`, `n_pairs`, `batch_size`, `seed`, `tags += ["cloned"]`). Cache key is `sha256(corpus) + n_pairs + batch_size + seed` compared against the existing `pack.json`; `force=True` bypasses. `session.clone_from_corpus(...)` is the thin public entry point wrapping this.
- `packs.py` ‚Äî `PackMetadata` + `ConceptFolder`. Shared `synthesize_pack_metadata(*, name, source, pack_dir, description="", tags=(), version="1.0.0", license="unknown", recommended_alpha=0.5, long_description="")` and `hash_folder_files(folder)` helpers used by both `hf.py::_install_synthesized_pack` (pack-less HF repos) and `session.py::_update_local_pack_files`. Divergent fields (version, license, description, source) are helper params, so each caller preserves its existing pack.json bytes on disk.
- `datasource.py` ‚Äî `DataSource` normalizes pairs from curated names, JSON, CSV, HF datasets, or raw lists.
- `results.py` ‚Äî `GenerationResult`, `TokenEvent`, `ProbeReadings` dataclasses with `to_dict()`. `TokenEvent`: `thinking`, `logprob`, `top_logprobs`, `finish_reason`. `GenerationResult`: `prompt_tokens`, `finish_reason` (`stop`/`length`/`stop_sequence`), optional `logprobs`. `ResultCollector` for batch export (dicts/JSONL/CSV/DataFrame).

### Generation loop
- `generation.py` ‚Äî Token-by-token + KV cache, wrapped in `torch.inference_mode()`. `GenerationConfig` is a **`@dataclass(frozen=True)`** ‚Äî callers rebind via `dataclasses.replace(session.config, temperature=...)` rather than mutating. In-flight generations snapshot config at entry and are immune to subsequent rebinds, so mid-flight `/temp` from the TUI or per-request overrides from the server don't affect running gens. Top-p via `torch.topk` (not full-vocab sort); `GenerationConfig.top_k` optional (default `None` = 1024-cap) is a hard cap on candidate pool applied **before** top-p (matches llama.cpp/Ollama order). In-place ops (`logits.div_()`, etc.) throughout. MPS sync at end-of-loop prevents Metal command-buffer reuse crashes. `generate_steered` accepts `seed`, `stop` (list), `logit_bias` (pre-built `(idx, val)` tensors), `presence_penalty`/`frequency_penalty` (per-token `completion_counts` dict, applied to raw logits before temperature), `logprobs` (0 = chosen-only, >0 = top-k via `log_softmax(logits.float())` before sampling; gated off hot path when `None`). `state.finish_reason` defaults to `length`, flips to `stop` on EOS/external-stop or `stop_sequence` on stop-string match (runs against accumulated non-thinking response text per emitted token; trims current emit to pre-stop prefix). **`None` done sentinel is NOT emitted by `generate_steered`** ‚Äî the session worker or TUI closure puts it on the queue in `finally`, so pending actions see the final state. `supports_thinking` delegates to `_detect_think_delimiters` which renders a round-trip assistant message through the chat template and returns `(start_id, end_id, response_start_id, starts_in_thinking)` ‚Äî works across families (Qwen `</think>`, Gemma `<channel|>`, gpt-oss `<|channel|>‚Ä¶<|message|>`) without hardcoded delimiter strings. `response_start_id` handles channel-based formats. **Explicit `ThinkingState` enum** on `GenerationState` (`IDLE ‚Üí PREAMBLE ‚Üí THINKING ‚Üí RESPONSE_PREAMBLE ‚Üí RESPONSE ‚Üí DONE`), set at every transition and forced to `DONE` in `finally`; additive ‚Äî `thinking_end_idx` remains the authoritative position marker. `\ufffd` partial-UTF-8 tokens marked `None` in the token table and buffered until a complete token, then flushed via `tokenizer.decode(pending_ids)`. `_get_token_table` uses **chunked (8192) `batch_decode`** with per-chunk try/except fallback to per-id `decode()` ‚Äî fast first-call on 150k+ vocabs. `_tok_key` shared cache-key helper for `_eos_cache`, `_token_table_cache`, `_think_delim_cache`. `GenerationState` = `stop_requested` (Event), `token_queue` (SimpleQueue), `thinking_end_idx`, `thinking_state`, `finish_reason`.

### TUI
- `tui/app.py` ‚Äî Thin frontend over `SaklasSession`. Owns local alpha/enabled/thinking state per panel, passes through per call. Thinking defaults ON for models that support it (`Ctrl+T` toggles). Polls at ~15 FPS. Commands: `/steer`, `/probe`, `/extract`, `/clear`, `/rewind`, `/sys`, `/temp`, `/top-p`, `/max`, `/help`. Mid-gen interruption: any conflicting action (Ctrl+R, new message, `/steer`, `/probe`, `/clear`, `/rewind`) stops current gen and defers via `_pending_action`; `_poll_generation` consumes the `None` sentinel and calls `_dispatch_pending_action` (single-site, try/except that resets `_gen_active` and surfaces errors via `add_system_message`). Panel focus uses index constants `_LEFT`/`_CHAT`/`_TRAIT`. **Per-token highlighting**: `Ctrl+Y` toggles, seeded from currently-selected probe in the trait panel; navigating the trait panel while on re-applies live via cached `self._assistant_messages` list. App reads `trait_panel.get_selected_probe()` on demand (no app-level `_highlight_probe` mirror ‚Äî each `_AssistantMessage` widget holds its own). The `_generate` worker **bypasses `session.generate_stream`** (to own its interruption/pending-action wiring) but uses the session's capture lifecycle: `session._begin_capture()` before `generate_steered`, `session._end_capture()` after, then `session.score_captured(generated)` ‚Äî same hook-driven path the API uses, no duplicate `HiddenCapture`. Per-probe arrays split at `thinking_end_idx`; batch-decode both halves in one `tokenizer.batch_decode`; push to captured widget via `call_from_thread(self._set_widget_token_data, ...)`.
- `tui/utils.py` ‚Äî `build_bar(value, max, width)` renders filled/empty bar pairs.
- `tui/vector_panel.py`, `chat_panel.py`, `trait_panel.py` ‚Äî Panels. **`chat_panel` is Markdown-free**: user + assistant messages are plain `Static` widgets with Rich-escaped text. `_AssistantMessage` holds exactly two content `Static`s (`#thinking-view`, `#response-view`) + a `Collapsible` wrapper for thinking. Tokens escaped once on append into `_escaped_chat_text`/`_escaped_thinking_text` (O(n) total). When per-token probe data arrives via `set_token_data`, a per-probe markup cache is pre-built with `_build_highlight_markup` so trait-panel navigation under highlight mode is a dict lookup, not a rebuild. Saturation fixed at `_HIGHLIGHT_SAT = 0.5` mapped to full `rgb(0..255, 0, 0)` / `rgb(0, 0..255, 0)`; zero scores get no background span (neutral text stays terminal-default). Thinking-block transitions via idempotent `ensure_thinking_collapsed()` (no `_in_thinking`, no `finalize()`, no `end_thinking()`). `trait_panel._nav_items` is `list[str]` (probe names directly).

### API server
- `server.py` ‚Äî FastAPI factory. OpenAI (`/v1/models`, `/v1/chat/completions`, `/v1/completions`) + saklas management (`/v1/saklas/vectors`, `/v1/saklas/probes`, `/v1/saklas/session`). Thin HTTP ‚Äî all routes call session with `stateless=True`. `_SamplingBase` pydantic shared by chat/completions: `stop`, `seed`, `logit_bias`, `presence_penalty`, `frequency_penalty`, `logprobs` (bool chat / int completions), `top_logprobs`, `stream_options.include_usage`, `max_completion_tokens` (aliased to `max_tokens` via model validator), accept-and-ignore for `user`, `n`, `response_format`, `messages[].name`. `ChatMessage._flatten_content` concatenates text parts of multimodal arrays (non-text rejected). Responses carry real `usage`, accurate `finish_reason` from `session._gen_state.finish_reason`, per-request `created`. `_stream_generation` emits first-chunk `{role: "assistant"}` delta; final chunk's `finish_reason` comes from gen state; optional usage chunk before `[DONE]` when `stream_options.include_usage`. Single `app.state.gen_lock` (`asyncio.Lock`) serializes non-streaming (`async with`) and streaming (held for full stream with 5-minute timeout ‚Üí 503) **across both OpenAI and Ollama routes**; requests queue FIFO rather than 409ing. Bearer auth via `SAKLAS_API_KEY` / `--api-key` applied as app-level dependency covering both protocols; unset = open. `RequestValidationError` handler maps pydantic errors to OpenAI shape. Thinking tokens stream as `reasoning_content` in chat delta. `_render_logprobs_chat` decodes token ids via tokenizer. Vector extraction streaming is synchronous (progress replayed after). `_gen_config_override` (in server.py) is the shared context manager both route families use for per-request `temperature`/`top_p`/`top_k`/`max_new_tokens` ‚Äî rebinds `session.config` via `dataclasses.replace` on entry, restores on exit. **`saklas/_server_runner.py`** holds the two helpers shared across OpenAI and Ollama routes: `acquire_lock_with_timeout(app)` (async context manager, 300s-bounded lock acquire, yields `bool` on timeout) and `run_blocking_generate(app, session, *, input, raw, gen_kwargs, temperature, top_p, max_tokens, top_k=None)` (`async with gen_lock` + `_gen_config_override` + `session.generate`). `_get_override_cm()` lazy-imports `_gen_config_override` from `server.py` to dodge a circular. `hf.py::_download` upgrades the error message when a user points at a dataset repo instead of a model repo (probed only on `RepositoryNotFoundError`). **Not supported (either protocol)**: tool calling, strict JSON/`json_schema` mode, embeddings.
- `ollama_api.py` ‚Äî Ollama-compatible shim mounted by `register_ollama_routes(app)` alongside OpenAI routes, reusing `session`/`default_alphas`/`gen_lock`/auth. Advertises `/api/version`, `/api/tags`, `/api/ps`, `/api/show`, `/api/chat`, `/api/generate`, `/api/pull` (no-op success for loaded model / 404 otherwise), + 501 stubs for `/api/push`, `/api/create`, `/api/copy`, `/api/delete`, `/api/embeddings`, `/api/embed`. NDJSON streaming (`application/x-ndjson`) matches Ollama wire format. `/api/show.template` reflects real HF Jinja `tokenizer.chat_template` (honest over useful-looking). `/api/generate` omits the `context` field (vs an empty list saklas can't round-trip). **Model aliasing** is hybrid: `_HF_TO_OLLAMA_ALIASES` overrides for names where Ollama's catalogue rounds differently (Gemma-2-2b is 2.6B but Ollama advertises `gemma2:2b`) or where `model_type` lacks a version suffix (Llama); `_infer_aliases(session)` falls back to `<family>:<size>` from `session.model_info` (`_normalise_family` strips `_text`/`_moe`/`forcausallm`; `_size_tag` rounds ‚â•10B to integer B, <10B keeps one decimal). Overrides win; inference fills gaps for new architectures. **Option translation** (`_resolve_options`): `temperature`, `top_p`, `top_k`, `seed`, `num_predict`‚Üí`max_tokens`, `stop`, `presence_penalty`, `frequency_penalty`, `repeat_penalty`, `steer`. `repeat_penalty` maps to `presence_penalty` via `ln(repeat_penalty)` ‚Äî equivalent to Ollama's "divide positive logits by penalty" (count-independent, bounded). Anything else in `options` (`min_p`, `mirostat*`, `num_ctx`, `typical_p`, ‚Ä¶) is logged at debug (`ollama: unsupported options dropped: ...`) and silently skipped. **Steering passthrough** via non-standard `steer` field inside `options` (or top-level): flat `{name: alpha}` or nested `{alphas: {...}, thinking: bool}`, merged over `default_alphas`, zero-alphas stripped. Clients that don't know about it pass through unchanged ‚Äî Open WebUI / Enchanted / LangChain's `ChatOllama` work out of the box. **Thinking** streams as `message.thinking` (chat) / top-level `thinking` (generate). **Strict mode**: `SAKLAS_STRICT_MODEL=1` makes `/api/chat` + `/api/generate` (and OpenAI `/v1/chat/completions` + `/v1/completions`) 404 when request `model` doesn't match loaded session by HF id or advertised alias; unset = accept any name. `_duration_stats` splits wall time proportionally between `prompt_eval_duration` + `eval_duration` in ns. `_finish_to_done_reason` maps saklas `stop_sequence` ‚Üí Ollama `stop`.
- `cli.py` ‚Äî Subcommand dispatch via the `_SUBCOMMAND_TABLE` dict (maps cmd ‚Üí `(parser_builder, runner)`). Per-subcommand parsers. `_print_startup(args)` shared between TUI and serve. `_warmup_session` rebinds `session.config` via `dataclasses.replace` (frozen config). `serve`: `--host/-H`, `--port/-P`, `--steer/-S name:alpha`, `--cors/-C`, `--api-key/-k`. Config loading via `-c` composes YAML files, auto-installs missing vectors, composed map lands on `args.config_vectors`.

## Performance rules

These gate `test_session.py::test_throughput` (steered ‚â• 85% of vanilla tok/s):

- **Hot-path hooks**: no Python allocation, no `.item()`, no CPU sync, in-place only. Norm preservation adds two `torch.linalg.vector_norm(dtype=float32)` calls + one in-place `mul_` per steered layer per step; fp32 accumulator avoids materializing an fp32 copy of the hidden tensor.
- **`torch.inference_mode()`** wraps the entire generation loop.
- **In-place ops**: `logits.div_()`, `logits.clamp_()`, `probs.div_()`, `hidden.add_()` / `mul_()`.
- **Top-p via `torch.topk`**, not full-vocab sort. `k = min(config.top_k or 1024, vocab)`; `top_k` is a hard cap applied before top-p (matches llama.cpp/Ollama).
- **Norms use fp32** ‚Äî fp16 sum-of-squares overflows at hidden_dim ‚â• 2048. Applies to extraction-time direction norms **and** the norm-preserving hook's pre/post captures.
- **Shares baked at extraction, flat scalar at apply.** Stored direction = `unit_direction * ref_norm * (score / sum(scores))`; hook math is `user_alpha * _STEER_GAIN * sum(baked)`. Per-unit-Œ± injection is invariant across layer count and absolute score magnitude. `_STEER_GAIN = 2.0` calibrated on gemma-4-31b-it with the bundled 21-probe pack; coherent band ‚âà Œ± 0.4‚Äì0.8, cliff ‚âà Œ± 1.0.
- **Norm preservation is unconditional.** Every steered layer's output rescaled back to pre-injection per-position magnitude. Keeps residual stream on its natural trajectory so high-Œ± rotation lands directly. *Net effect*: the v1.3.1 sweep confirmed norm preservation made injection **more** efficient per unit Œ± (not less as you'd naively expect) ‚Äî unconstrained addition also inflates magnitude which implicitly attenuates directional change. Same `_STEER_GAIN = 2.0` survived the regime change.
- **Monitor capture is hook-driven, inline with generation.** `HiddenCapture` hooks union of probe layers for the run; `score_per_token` scores from captured tensors afterward ‚Äî no second forward pass. Per-layer weight = `||baked||` (= share √ó ref_norm). Mean-centered cosine sims, bounded [-1, 1]. Scoring uses a **per-layer stacked cache** (`V[P,D]`, `W[P]`) so one matmul per layer scores all probes simultaneously; only one `.cpu().tolist()` per measure regardless of probe count.
- **Steering hooks are transient** ‚Äî composed before generation, removed after.
- **Contrastive diffs in float32** ‚Äî fp16 subtraction between close vectors loses precision.
- **MPS discipline** ‚Äî diffs kept on CPU (SVD runs there anyway), `torch.mps.empty_cache()` between extraction passes, `torch.device(target)` model loading to avoid unified-memory RSS spike.

## Bundled statements

21 curated concepts at **n=45 pairs each**, stored in `saklas/data/vectors/<canonical>/statements.json`. 19 bipolar (`happy.sad`, `masculine.feminine`, `high_context.low_context`, ‚Ä¶) + 2 monopolar (`agentic`, `manipulative`). Full list: see `saklas/data/vectors/`.

**Unified shared-scenario embody framework.** Both modes use the same prompt skeleton: each pair anchors both speakers in the same concrete situation drawn from `_SCENARIO_BANK` (60 broad situations covering work/home/social/public/solo/intellectual/charged contexts) and asks each speaker to *embody* one pole of the axis in voice, diction, and rhythm. Cross-pair variance is situational, within-pair variance is pure pole ‚Äî top PCA component locks onto pole not topical cluster. Positively framed (no blocklists), explicitly requests voice/register contrast (not just framing differences), licenses stereotyped/hammy output where the pole has a natural register. Bipolar names both poles; monopolar asks the model to embody the *semantic opposite* of the concept (prior `Speaker B is unrelated` baseline produced noise pairs that extracted a concept-vs-noise direction).

**Allegorical embodiment is by design.** Steering vectors capture voice/register shifts on the model's own linguistic output, so non-human concepts get projected onto human expressive modes ‚Äî a `deer.wolf` probe yields timid-person voice vs predatory-person voice, not literal sensory experience. This is the only form a LM steering vector can take; the metaphor lens is what makes abstract concepts (`bacterium.virus`, `the_sun.the_moon`, `brick.feather`) extract clean axes via the same pipeline.

90 neutral statements in `saklas/data/neutral_statements.json` follow the same affect-neutral, topically-diverse discipline. All generated via `scripts/regenerate_bundled_statements.py` on gemma-4-31b-it. The script reads an explicit `BIPOLAR`/`MONOPOLAR` manifest (not a folder walk) so concept identity is authoritative; `--purge` wipes the data dir before regen.

**Canonical naming rule.** `session.canonical_concept_name(concept, baseline=None)` slugs poles via `[^a-z0-9]+ ‚Üí _` (`high-context` ‚Üí `high_context`) and joins bipolar via `BIPOLAR_SEP = "."`. `session.extract()` returns `(canonical_name, profile)`. `/steer happy - sad` and `/steer happy.sad` resolve to the same vector. `NAME_REGEX = ^[a-z][a-z0-9._-]{0,63}$`; `.` and `_` allowed, `@` forbidden (HF revision separator). `.` chosen over `~` because HF Hub repo names reject `~`.

**Pole aliasing.** `cli_selectors.resolve_pole(raw, namespace=None)` scans `_all_concepts()` across every namespace, returns `(canonical, sign, match)`. Typing a bare pole (e.g. `wolf`) on top of an installed `bob/deer.wolf` returns `("deer.wolf", -1, ‚Ä¶)`; caller multiplies user Œ± by `sign`. Monopolar exact matches and composite literals pass through unchanged. Ambiguity ‚Üí `AmbiguousSelectorError` (disambiguate with `ns/name`). Explicit `/steer pos - neg` skips resolution so user-declared poles always win.

Pair-count note: <n=32 inflates mean PCA scores ~38% (small-sample bias), which used to destabilize the old `_REF_SCORE / mean_score` normalization. Share-baking is mean-magnitude-invariant so small-sample bias no longer breaks cross-concept Œ± consistency ‚Äî but n=45 remains the measured sweet spot for axis quality (no layer-direction flips, ~25% faster extraction than n=60).
