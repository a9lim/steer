# CLAUDE.md

## What this is

`saklas` is a Python library + Textual TUI + dual-protocol HTTP server (OpenAI `/v1/*` **and** Ollama `/api/*` on the same port) for activation steering and trait monitoring on HuggingFace causal LMs. Representation Engineering via contrastive PCA; per-call alpha control; no model mutation. Three frontends over one engine: `SaklasSession` (programmatic), `saklas serve` (HTTP), `saklas` (TUI).

Version `1.4.1`. Lives in **two places** — bump both or `saklas.__version__` drifts from PyPI:
- `pyproject.toml` → `[project] version`
- `saklas/__init__.py` → `__version__`

Releases: merge bump to `main` → `.github/workflows/release.yml` tags `v$VERSION`, builds, publishes via trusted publishing, cuts a GitHub release. Push without bumping → no-op.

## Commands

```bash
pip install -e ".[dev]"                # editable + pytest
pip install -e ".[serve]"              # fastapi + uvicorn
pip install -e ".[gguf]"               # llama.cpp GGUF I/O
pip install -e ".[cuda]"               # bitsandbytes + flash-attn (Linux/CUDA)
saklas tui <model_id>                          # TUI (explicit subcommand)
saklas serve <model_id>                         # OpenAI + Ollama API (dual-protocol)
saklas pack install <target> [-s|-a NS/N|-f]    # HF coord or folder; -s = statements only
saklas pack refresh <selector> [-m MODEL]       # re-pull; `refresh neutrals` is reserved
saklas pack clear <selector> [-m MODEL]         # delete per-model tensors (keep statements)
saklas pack rm <selector> [-y]                  # remove folder (bundled respawns)
saklas pack ls [selector] [-j|-v]               # LOCAL installed packs only
saklas pack search <query> [-j|-v]              # search HF hub for saklas-pack repos
saklas pack push <selector> [-a OWNER/NAME] [-p] [-m MODEL] [-s|-n] [-t] [-d] [-f]
saklas pack export gguf <selector> [-m MODEL] [-o PATH] [--model-hint HINT]
saklas vector extract <concept>|<pos> <neg> [-m MODEL] [-f]
saklas vector merge <name> <components> [-m]    # merge baked tensors
saklas vector clone <corpus> -N NAME [-m MODEL] [-n N_PAIRS] [--seed S] [-f]
saklas vector compare <concepts...> -m MODEL [-v] [-j]       # cosine similarity between vectors
saklas config show [-c PATH ...] [--no-default] [-m MODEL]   # print effective merged config
saklas config validate <file>                                # exit 0 valid / 2 invalid
pytest tests/                                   # all; GPU tests gated on CUDA/MPS
```

The root parser in `cli.py` has exactly five verbs — `tui`, `serve`, `pack`, `vector`, `config`. `pack` nests 8 distribution verbs (`install/refresh/clear/rm/ls/search/push/export`), `vector` nests 4 computation verbs (`extract/merge/clone/compare`), `config` nests `show` and `validate`. No `argv[0]` peeking, no top-level verb aliases, no bare-TUI fallback — `saklas google/gemma-2-2b-it` is an argparse error. Bare `saklas` prints help + hint and exits 0. Bare `saklas pack` / `saklas vector` / `saklas config` print their own verb list and exit 0.

Every subcommand that takes `-c/--config` auto-loads `~/.saklas/config.yaml` first, then composes any explicit `-c` files on top (later overrides earlier). `ConfigFile.effective(extras, include_default=...)` is the single entry point; `ConfigFile.resolve_poles()` runs every `vectors:` key through `cli_selectors.resolve_pole` so bare poles (`wolf: 0.5`) land as canonical + signed (`deer.wolf: -0.5`) in `default_alphas`.

Selector grammar (shared): `<name>`, `<ns>/<name>`, `tag:<t>`, `namespace:<ns>`, `default`, `all`. Bare names resolve cross-namespace and raise `AmbiguousSelectorError` on collision. `pack refresh` silently skips `source=local`; `pack refresh neutrals` rewrites `~/.saklas/neutral_statements.json` from the bundled copy. `pack rm` refuses broad selectors without `-y`; bundled concepts respawn on next session init. `pack ls` is local-only; HF hub search moved to the dedicated `pack search` verb.

## Python API shape (v2.0)

```python
from saklas import SaklasSession, SamplingConfig, Steering, Profile
from saklas import SaklasError, EventBus, SteeringApplied, ProbeScored

with SaklasSession.from_pretrained("google/gemma-3-4b-it", device="auto") as session:
    name, profile = session.extract("angry.calm")     # Profile, not dict
    session.steer(name, profile)                      # register by name

    # Per-call sampling + steering. Both are frozen dataclasses; None = use session defaults.
    result = session.generate(
        "What makes a good day?",
        steering={name: 0.3},                          # or Steering(alphas=..., thinking=...)
        sampling=SamplingConfig(temperature=0.7, max_tokens=256, seed=42),
        thinking=None,                                 # None = auto via supports_thinking
    )

    # Context-manager form: scoped, nested, canonical pole-resolution site.
    with session.steering({"wolf": 0.5}):              # resolves to deer.wolf @ -0.5
        result = session.generate("Describe a forest.")

    # Streaming with public callback (cluster 3).
    for tok in session.generate_stream("Tell me a story.", steering={name: 0.2}):
        print(f"[think] {tok.text}" if tok.thinking else tok.text, end="", flush=True)

    # Event bus.
    session.events.subscribe(lambda e: print("event:", type(e).__name__))
```

Hard-break notes vs v1.4:
- `generate` and `generate_stream` are **keyword-only** past `input`. All sampling kwargs (seed/stop/logit_bias/penalties/logprobs/temperature/top_p/top_k/max_tokens) live on `SamplingConfig`; the v1 `alphas=` kwarg is gone — use `steering=`.
- `extract()` returns `(name, Profile)`, not `(name, dict)`. The dict-compat surface on `Profile` (`__getitem__` / `__iter__` / `len` / `items`) means most v1 loop code still works.
- `SaklasSession.__init__` now takes a pre-loaded `PreTrainedModel`; construct with `SaklasSession.from_pretrained(model_id, ...)`.
- `session.config = replace(session.config, temperature=0.8)` still works for session-level defaults (`GenerationConfig` is trimmed to session-level fields only); **per-call overrides should use `SamplingConfig`**, not rebind `session.config`.
- `session.lock` is an `asyncio.Lock` owned by the server layer. Library-only callers never touch it.
- Every saklas-raised exception is a `SaklasError` subclass while preserving its stdlib MRO, so `except SaklasError` catches the whole family and `except ValueError` / `except RuntimeError` at existing sites still works.

## Cache layout

All state under `~/.saklas/` (override via `SAKLAS_HOME`):

```
~/.saklas/
  neutral_statements.json              # user-editable (copy-on-miss from package)
  vectors/
    default/<concept>/                 # bundled (source=bundled)
      pack.json                        # name, description, tags, files{sha256}
      scenarios.json                   # {"scenarios": [...]}; 9 broad domains used as pair-gen seed
      statements.json                  # contrastive pairs
      <safe_model_id>.safetensors      # baked tensor (native)
      <safe_model_id>.json             # slim sidecar: method/saklas_version/statements_sha256
      <safe_model_id>.gguf             # optional llama.cpp parallel
    local/<concept>/                   # user-authored (source=local; `refresh` skips)
    <hf_owner>/<concept>/              # HF-pulled (source=hf://owner/name[@rev])
  models/<safe_model_id>/
    layer_means.safetensors            # probe-centering baseline
    layer_means.json                   # hash of neutral_statements.json
```

- `pack.json.files` sha256 map verified on every `ConceptFolder.load`.
- Slim sidecars — shares are baked into tensor magnitudes at extraction, so no `scores` field anymore.
- **Dual-format tensors**: `ConceptFolder` globs `*.safetensors` + `*.gguf`. Safetensors wins on same-stem conflict (native, carries sidecar). GGUF metadata lives in the header, no external sidecar — `sidecar(sid)` raises `KeyError` for GGUF-only entries. `vectors.load_profile` dispatches on extension; both paths return `(profile, metadata)`.
- **Upgrade gotcha**: `materialize_bundled` is copy-on-miss for `neutral_statements.json` and for missing concept folders, **but auto-upgrades bundled concepts in place when the on-disk `pack.json.format_version` is explicitly stale** (v1.x users upgrading to v2.0 had format_version=1 packs that the new reader rejects; we overwrite shipped `pack.json` + `statements.json` while leaving any per-model tensor files intact). Releases changing neutrals still need `saklas pack refresh neutrals`; layer means auto-recompute via hash check in `bootstrap_layer_means`.

**HF distribution.** Packs are **model repos** (not datasets) — safetensors is model-hub-native and `base_model` frontmatter gives reverse-link discoverability. `saklas/hf.py` uses `repo_type="model"` exclusively. `saklas install owner/name@rev` pins to any git ref (tag/branch/SHA); `hf.split_revision` threads `revision` through download/refresh so pinning is pinning.

**Pack-less install.** `install owner/name[@rev]` synthesizes `pack.json` when the HF repo has no root manifest: scans `*.safetensors`/`*.gguf`, fabricates `method="imported"` sidecars for bare safetensors, computes hashes, slugs `name` from the repo name via `NAME_REGEX`, sets `source = "hf://..."`. Repeng-style GGUF-only repos (`jukofyork/creative-writing-control-vectors-v3.0`, etc.) install with zero prep. Subsequent ops treat the folder identically to native packs; `push` still requires `-f` + `-a` (same guard as pulled HF packs).

**`saklas push`.** Publishes one concept folder as an HF model repo. Default coord `<whoami>/<pack_name>`; `--as/-a` overrides. `-p` private, `-m MODEL` scope tensors, `-s` statements-only, `-n` tensors-without-statements, `-t` tag `v<pack.version>`, `-d` dry-run (stages README/.gitattributes/filtered pack.json), `-f` force-through for non-owned packs. Model card YAML carries `library_name: saklas`, merged tags (`saklas-pack`, `activation-steering`, `steering-vector`, + `pack.tags`), a `base_model:` list from sidecar filenames, `base_model_relation: adapter`. `.gitattributes` pins `*.safetensors` to LFS.

**`saklas export gguf`.** Writes baked tensors to llama.cpp's control-vector GGUF format. Wire: `general.architecture = "controlvector"`, `controlvector.model_hint = <arch>` (via `transformers.AutoConfig.model_type`; `--model-hint` overrides), `controlvector.layer_count`, tensors `direction.<layer_idx>` as fp32. Because shares are baked, llama.cpp's uniform `--control-vector-scaled` scalar reproduces saklas's per-layer weighting for free — no per-layer metadata needed. **Bundled concepts refuse in-place export** (their folder is restored on `refresh`; any in-place GGUF would silently vanish) — pass `-o` pointing outside the pack folder.

## Architecture

Five layers: **model/vector**, **steering/monitoring**, **session**, **TUI**, **server**. v2.0 package layout is `saklas/{core,io,cli,server,tui}/` with `saklas/__init__.py` pinning the public re-exports (`SaklasSession`, `SaklasError`, `Profile`, `ProfileError`, `SamplingConfig`, `Steering`, `EventBus` + event dataclasses, `DataSource`, `GenerationResult`, `TokenEvent`, `ProbeReadings`, `ResultCollector`). Consumers that do `from saklas import X` keep working; anything reaching into private submodule paths is expected to break (hard-break refactor).

### Model + vector
- `core/model.py` — HF causal LM loading. `_LAYER_ACCESSORS` maps `model_type` → layer-list accessor (`def`, not lambdas). **Tested and known working** (= `_TESTED_ARCHS` frozenset): `qwen2`, `qwen3`, `gemma2`, `gemma3`, `mistral3`, `gpt_oss`, `llama`, `glm`. **Wired up but untested**: mistral/mixtral, phi/phi3/phimoe, cohere 1–2, deepseek v2/v3, starcoder2, olmo 1–3 + moe, granite/granitemoe, nemotron, stablelm, gpt2/neo/j/bigcode/neox, bloom, falcon/falcon_h1, mpt, dbrx, opt, recurrent_gemma. `get_layers()` emits a one-time `UserWarning` (gated by `_warned: set[str]`) when a loaded model's `model_type` isn't in `_TESTED_ARCHS` — table still resolves, user just gets a flag. Be careful claiming breadth in user-facing docs — the accessor table being 54 entries is not the same as 54 architectures working. Adding a new architecture = one entry. Cascading fallbacks on attention impl (SDPA → eager), dtype, device. `_load_text_from_multimodal` extracts text-only sub-models (Ministral-as-Mistral3), strips `language_model.` prefixes, dequantizes FP8 inline. Patches `torch.histc` for MPS MoE routing.
- `core/vectors.py` — One forward pass per prompt; `_capture_all_hidden_states` hooks every layer in a single pass. `_encode_and_capture_all` pools from the **last content token**, walking back past `tokenizer.all_special_ids` to skip trailing chat-template markers (Llama `<|eot_id|>`, Gemma `<end_of_turn>`, Qwen `<|im_end|>`). Contrastive diffs cast to **fp32** before differencing (fp16 loses precision on close vectors, makes SVD degenerate). Multi-pair: batched SVD across all layers (one `torch.linalg.svd` call on stacked `(n_layers, N, dim)`), scored by explained-variance ratio. Single-pair: scored by `diff_norm / activation_norm`. **Shares baked at extraction**: each direction scaled to mean activation norm of its layer, then multiplied by `score_i / sum(scores)`; hook-side math collapses to flat `user_alpha * _STEER_GAIN * sum(baked)` (all-zero fallback to uniform). Returns a **profile**: `dict[int, Tensor]` covering every layer — no scores dict. `compute_layer_means` averages 90 neutral prompts for centering. `load_profile` dispatches on extension. `_template_overhead_cache` is keyed by `id(tokenizer)` — safe only while one tokenizer lives the session lifetime.
- `io/gguf_io.py` — `write_gguf_profile(profile, path, *, model_hint)` + `read_gguf_profile(path)`. Validates `controlvector` architecture field, requires `model_hint`, parses `direction.*` tensor names for layer indices, returns `(profile, {method: "gguf_import", ...})`. Lazy import of `gguf`; raises `GGUFNotInstalled` (ImportError subclass) with install hint. Repeng unit-normed GGUFs round-trip fine; saklas-baked GGUFs carry share-weighted magnitudes.
- `io/probes_bootstrap.py` — Walks `~/.saklas/vectors/default/` after `materialize_bundled()`. 21 bundled probes (19 bipolar + 2 monopolar; see ``regenerate_bundled_statements.py`` manifest for the authoritative list) categorized via `pack.json` tags (`affect`, `epistemic`, `alignment`, `register`, `social_stance`, `cultural`). `bootstrap_layer_means` caches `~/.saklas/models/<id>/layer_means.safetensors`, stale when `neutral_statements.json` hash changes. MPS cache flushed between probes.

### Steering + monitoring
- `core/hooks.py` — `SteeringHook` adds a pre-composed vector to hidden states and **unconditionally rescales each position back to its pre-injection norm** (in-place, `torch.linalg.vector_norm(dtype=float32)` to avoid fp16 sum-of-squares overflow at hidden_dim ≥ 2048). No flag, no escape hatch. Prevents crank-alpha-gibberish by keeping the residual stream on its natural magnitude trajectory while still rotating toward the pole. `SteeringManager` groups vectors by layer, sums co-layer directions into **one pre-composed hook per active layer**. `add_vector(name, profile, alpha)`. **Flat-scalar apply**: `effective_injection = user_alpha * _STEER_GAIN * sum(baked)` — no per-layer score lookup, no `sum(scores)` division. Preserves **layer-count invariance** (different depths hit same behavior at same α) and **score-magnitude invariance** (absolute PCA scores don't matter, only relative shares) — both just moved from apply-time to extract-time. `_STEER_GAIN = 3.5` calibrated on gemma-4-31b-it with the open-ended scenario framework; α-sweep on `angry.calm` post-norm-preservation: coherent band α ≈ 0.2–0.6, cliff α ≈ 0.75. Cross-model validation on Qwen3.5-9B and Llama-3.2-3B-Instruct confirms the cliff transfers within ±0.1α across architectures. Smaller/MoE/heavily-safety-trained models may still need proportionally higher α for residual architectural effects this doesn't capture. `HiddenCapture` is the companion class used by session + TUI: `attach(layers, layer_indices)` / `detach()` / `stacked()` — each capture is `detach().clone()` of `output[0, -1, :]` (device-local, no sync); k-th capture = "state that produced generated token k".
- `core/monitor.py` — `TraitMonitor` scores probes against per-layer hidden states via `_score_probes` (shared `_normalize_hidden` helper mean-centers with layer means, L2-normalizes, magnitude-weighted cosine sim). **Per-layer weight = `||baked||`** (= `share_L * ref_norm_L`) — recovers the "how much does this layer read per unit α" quantity from the tensor magnitude itself. `_ensure_cache` builds a **per-layer stacked cache** `{layer_idx: (V[P,D], W[P])}` where `V` holds unit-normed probe directions (zero rows for probes missing that layer) and `W[p]` = `||baked_p_L||` (0 when missing). One matmul per layer scores every probe against a hidden state in a single kernel launch; a single `.cpu().tolist()` at the end pulls the final aggregate out — the scoring hot path has **zero `.item()` calls** regardless of probe count. Entry points: `score_per_token(captured, generated_ids, tokenizer, *, accumulate)` is primary — takes per-layer `(n_tokens, dim)` captures, returns `(aggregate_vals, per_token_scores)`, updates history when `accumulate=True`. Aggregate pools from the last non-special token. `measure_from_hidden(hidden_per_layer)` is the test-path pre-aggregated entry. `measure(model, tokenizer, layers, text)` still runs a forward pass — kept as a convenience for scoring arbitrary text outside a generation run. `_pending_aggregate` / `_pending_per_token` split so the TUI poll sees aggregate readiness independently. `add_probe`/`remove_probe` invalidate the stacked cache wholesale (rebuilt lazily next score). `probe_names`, `profiles`, `layer_means` are properties (no defensive copies).

### Session
- `core/session.py` — `SaklasSession` owns model, profile registry (`_profiles`), monitor, `SteeringManager`, `HiddenCapture`, generation defaults (`session.config`, frozen `GenerationConfig`), conversation history, and a synchronous `events: EventBus`. **Construction** via `SaklasSession.from_pretrained(model_id, *, device, dtype, quantize, probes, ...)` classmethod (does HF load + probe bootstrap + layer-mean compute); plain `__init__(model, tokenizer, *, probes, ...)` accepts a pre-loaded `PreTrainedModel` for multi-session-on-one-model scenarios. **Public API on `generate` / `generate_stream`** is keyword-only:
  ```python
  session.generate(input, *, steering=None, sampling=None, stateless=False, raw=False, thinking=None, on_token=None)
  ```
  `steering` accepts a `Steering` or a bare `dict[str, float]`; `sampling` is a `SamplingConfig` (frozen dataclass mirroring `temperature`/`top_p`/`top_k`/`max_tokens`/`seed`/`stop`/`logit_bias`/`presence_penalty`/`frequency_penalty`/`logprobs`); `thinking=None` auto-detects via `supports_thinking`; `on_token` is a public callback. **`session.steering(alphas)` context manager** is the canonical pole-resolution site (pushes onto a LIFO stack, resolves pole aliases via `cli.selectors.resolve_pole`, rebuilds hooks from the flattened head; nesting flattens with inner-wins semantics; emits `SteeringApplied` / `SteeringCleared` events). `_resolve_pole_aliases` has a **cache-hit auto-load fast path** (`_try_autoload_vector`) — if a referenced concept name isn't in `_profiles` but has an installed pack with an already-extracted per-model tensor on disk, the tensor is loaded inline so HTTP clients can steer bundled probes without a pre-registration dance. No PCA, no network. **No persistent steering hooks.** **`generate` and `generate_stream` are thin wrappers around `_generate_core`**, which owns the `_gen_lock` threading re-entry guard (flips `_gen_active`), the steering context, `_begin_capture`/`_end_capture`, `_finalize_generation`, and the deterministic teardown order (`stop_requested.set()` → worker join where applicable → `_end_capture()` → `_clear_steering()`). `generate_stream` spawns a worker running `_generate_core` with an internal `on_token` that enqueues `TokenEvent`s into a local `SimpleQueue`; iterator `close()` / `GeneratorExit` triggers the same teardown path. **Monitor scoring is in-flight**: `_generate_core` attaches `HiddenCapture` on the union of probe layers, `_finalize_generation` → `score_captured` → `monitor.score_per_token`, no second forward pass. **Events emitted** from the hot path: `GenerationStarted`, `SteeringApplied`, `SteeringCleared`, `ProbeScored`, `GenerationFinished`, plus `VectorExtracted` from `extract()`. Server/TUI subscribers must hop via `loop.call_soon_threadsafe` if they need an event-loop context. **`session.lock`** is a separate `asyncio.Lock` (distinct from the threading `_gen_lock`); it's the server-owned async-level serializer that queues concurrent HTTP requests FIFO, not a generation-reentry guard. `last_per_token_scores` and `last_result` are public properties. **Regime note**: scoring uses actual generation-regime hidden states with full prompt context; `layer_means` are still computed wrap-assistant — mean-centering is slightly off-regime but neutral drift is small, recompute via `saklas pack refresh neutrals` if you see bias. `_generation_preamble` dedupes lock/steering/preamble setup. Full custom-concept pipeline: tensor cache → curated or local `statements.json` (reused by default) → generate scenarios → save `scenarios.json` → generate pairs → save `statements.json` → contrastive PCA → save tensor. Curated concepts save under `default/<c>/`; user concepts under `local/<c>/`. Scenario and statement caches are model-independent. `_update_local_pack_files` refreshes `pack.json.files` via the shared `hash_folder_files` helper in `io/packs.py`. `_N_PAIRS = 45`. `canonical_concept_name(concept, baseline=None)` slugs poles via `[^a-z0-9]+ → _` and joins bipolar via `BIPOLAR_SEP = "."`; `extract()` returns `(canonical_name, Profile)` (cluster 2 promoted the return type from the raw `dict[int, Tensor]`). Extract lookup scans `saklas.cli.selectors._all_concepts()` across every namespace, so `/steer bob/deer.wolf` hits HF-pulled packs without being bundled.
- `core/sampling.py` / `core/steering.py` / `core/events.py` / `core/errors.py` / `core/profile.py` — Per-call `SamplingConfig` (frozen with `merged_with`), `Steering` (frozen, `from_value` coerces dict/None), synchronous `EventBus` + 6 event dataclasses, `SaklasError` base (all other saklas exceptions multi-inherit through it while preserving their stdlib MRO), and the `Profile` class wrapping `dict[int, Tensor]` with `.layers/.weight_at/.save/.load/.to_gguf/.merged/.merged_with/.promoted_to/.cosine_similarity`. `cosine_similarity(other, *, per_layer=False)` computes magnitude-weighted cosine over shared layers (aggregate `float`) or raw per-layer cosines (`dict[int, float]`). Empty intersection raises `ProfileError`. All re-exported from `saklas` top-level.
- `io/cloning.py` — Training-free persona cloning. `clone_from_corpus(session, path, name, *, n_pairs=90, seed=None, batch_size=5, force=False)` reads a one-utterance-per-line text file, samples `n_pairs` exemplars, and pairs each against a *neutralized* rewrite generated by the same model in batches of `batch_size` (default `_BATCH_SIZE = 5` — larger models can take bigger batches, small/MoE may need smaller). **Delegates to `session.extract(DataSource(pairs=...))`** for the contrastive PCA + save — cloning owns only corpus hashing, exemplar sampling, rewrite batching, and the cache short-circuit. Result is saved to `local/<name>/` and then `pack.json` is overwritten with clone-specific metadata (`corpus_sha256`, `n_pairs`, `batch_size`, `seed`, `tags += ["cloned"]`). Cache key is `sha256(corpus) + n_pairs + batch_size + seed` compared against the existing `pack.json`; `force=True` bypasses. `session.clone_from_corpus(...)` is the thin public entry point wrapping this.
- `io/packs.py` — `PackMetadata` + `ConceptFolder`. `PACK_FORMAT_VERSION = 2`; loader raises `PackFormatError` on explicit v1 with a `scripts/upgrade_packs.py` hint. `materialize_bundled` detects stale v1 on bundled concepts and upgrades in place. Shared `synthesize_pack_metadata(*, name, source, pack_dir, description="", tags=(), version="1.0.0", license="unknown", recommended_alpha=0.5, long_description="")` and `hash_folder_files(folder)` helpers used by both `hf.py::_install_synthesized_pack` (pack-less HF repos) and `session.py::_update_local_pack_files`. Divergent fields (version, license, description, source) are helper params, so each caller preserves its existing pack.json bytes on disk.
- `io/datasource.py` — `DataSource` normalizes pairs from curated names, JSON, CSV, HF datasets, or raw lists.
- `core/results.py` — `GenerationResult`, `TokenEvent`, `ProbeReadings` dataclasses with `to_dict()`. `TokenEvent`: `thinking`, `logprob`, `top_logprobs`, `finish_reason`. `GenerationResult`: `prompt_tokens`, `finish_reason` (`stop`/`length`/`stop_sequence`), optional `logprobs`. `ResultCollector` for batch export (dicts/JSONL/CSV/DataFrame).

### Generation loop
- `core/generation.py` — Token-by-token + KV cache, wrapped in `torch.inference_mode()`. `GenerationConfig` is a **`@dataclass(frozen=True)`** holding session-level defaults (`max_new_tokens`, `temperature`, `top_p`, `top_k`, `system_prompt`); callers rebind session defaults via `dataclasses.replace(session.config, ...)` rather than mutating. **Per-call sampling overrides never touch `session.config`** — `_generate_core` composes a local `GenerationConfig` from `session.config` merged with the call's `SamplingConfig` via `_compose_gen_config(sampling)`, and hands that to `generate_steered`. That means an in-flight generation is immune to concurrent session-default rebinds (`/temp` from the TUI doesn't affect the running gen; per-request overrides from the server live on the call, not on `session.config`). Top-p via `torch.topk` (not full-vocab sort); `GenerationConfig.top_k` optional (default `None` = 1024-cap) is a hard cap on candidate pool applied **before** top-p (matches llama.cpp/Ollama order). In-place ops (`logits.div_()`, etc.) throughout. MPS sync at end-of-loop prevents Metal command-buffer reuse crashes. `generate_steered` accepts `seed`, `stop` (list), `logit_bias` (pre-built `(idx, val)` tensors), `presence_penalty`/`frequency_penalty` (per-token `completion_counts` dict, applied to raw logits before temperature), `logprobs` (0 = chosen-only, >0 = top-k via `log_softmax(logits.float())` before sampling; gated off hot path when `None`). `state.finish_reason` defaults to `length`, flips to `stop` on EOS/external-stop or `stop_sequence` on stop-string match (runs against accumulated non-thinking response text per emitted token; trims current emit to pre-stop prefix). **`None` done sentinel is NOT emitted by `generate_steered`** — the session worker or TUI closure puts it on the queue in `finally`, so pending actions see the final state. `supports_thinking` delegates to `_detect_think_delimiters` which renders a round-trip assistant message through the chat template and returns `(start_id, end_id, response_start_id, starts_in_thinking)` — works across families (Qwen `</think>`, Gemma `<channel|>`, gpt-oss `<|channel|>…<|message|>`) without hardcoded delimiter strings. `response_start_id` handles channel-based formats. **Explicit `ThinkingState` enum** on `GenerationState` (`IDLE → PREAMBLE → THINKING → RESPONSE_PREAMBLE → RESPONSE → DONE`), set at every transition and forced to `DONE` in `finally`; additive — `thinking_end_idx` remains the authoritative position marker. `\ufffd` partial-UTF-8 tokens marked `None` in the token table and buffered until a complete token, then flushed via `tokenizer.decode(pending_ids)`. `_get_token_table` uses **chunked (8192) `batch_decode`** with per-chunk try/except fallback to per-id `decode()` — fast first-call on 150k+ vocabs. `_tok_key` shared cache-key helper for `_eos_cache`, `_token_table_cache`, `_think_delim_cache`. `GenerationState` = `stop_requested` (Event), `token_queue` (SimpleQueue), `thinking_end_idx`, `thinking_state`, `finish_reason`.

## Performance rules

These gate `test_session.py::test_throughput` (steered ≥ 85% of vanilla tok/s):

- **Hot-path hooks**: no Python allocation, no `.item()`, no CPU sync, in-place only. Norm preservation adds two `torch.linalg.vector_norm(dtype=float32)` calls + one in-place `mul_` per steered layer per step; fp32 accumulator avoids materializing an fp32 copy of the hidden tensor.
- **`torch.inference_mode()`** wraps the entire generation loop.
- **In-place ops**: `logits.div_()`, `logits.clamp_()`, `probs.div_()`, `hidden.add_()` / `mul_()`.
- **Top-p via `torch.topk`**, not full-vocab sort. `k = min(config.top_k or 1024, vocab)`; `top_k` is a hard cap applied before top-p (matches llama.cpp/Ollama).
- **Norms use fp32** — fp16 sum-of-squares overflows at hidden_dim ≥ 2048. Applies to extraction-time direction norms **and** the norm-preserving hook's pre/post captures.
- **Shares baked at extraction, flat scalar at apply.** Stored direction = `unit_direction * ref_norm * (score / sum(scores))`; hook math is `user_alpha * _STEER_GAIN * sum(baked)`. Per-unit-α injection is invariant across layer count and absolute score magnitude. `_STEER_GAIN = 3.5` calibrated on gemma-4-31b-it with the bundled 21-probe pack; coherent band ≈ α 0.2–0.6, cliff ≈ α 0.75.
- **Norm preservation is unconditional.** Every steered layer's output rescaled back to pre-injection per-position magnitude. Keeps residual stream on its natural trajectory so high-α rotation lands directly. *Net effect*: the v1.3.1 sweep confirmed norm preservation made injection **more** efficient per unit α (not less as you'd naively expect) — unconstrained addition also inflates magnitude which implicitly attenuates directional change. Same `_STEER_GAIN = 3.5` survived the regime change.
- **Monitor capture is hook-driven, inline with generation.** `HiddenCapture` hooks union of probe layers for the run; `score_per_token` scores from captured tensors afterward — no second forward pass. Per-layer weight = `||baked||` (= share × ref_norm). Mean-centered cosine sims, bounded [-1, 1]. Scoring uses a **per-layer stacked cache** (`V[P,D]`, `W[P]`) so one matmul per layer scores all probes simultaneously; only one `.cpu().tolist()` per measure regardless of probe count.
- **Steering hooks are transient** — composed before generation, removed after.
- **Contrastive diffs in float32** — fp16 subtraction between close vectors loses precision.
- **MPS discipline** — diffs kept on CPU (SVD runs there anyway), `torch.mps.empty_cache()` between extraction passes, `torch.device(target)` model loading to avoid unified-memory RSS spike.

## Bundled statements

21 curated concepts at **n=45 pairs each** (9 scenarios × 5 pairs/scenario), stored in `saklas/data/vectors/<canonical>/` with `scenarios.json` + `statements.json`. 19 bipolar + 2 monopolar (`agentic`, `manipulative`). Authoritative list lives in the `BIPOLAR`/`MONOPOLAR` manifest in `scripts/regenerate_bundled_statements.py`. Categories: `affect` (angry.calm, happy.sad), `epistemic` (confident.uncertain, honest.deceptive, hallucinating.grounded), `alignment` (refusal.compliant, sycophantic.blunt, agentic, manipulative), `register` (formal.casual, direct.indirect, verbose.concise, creative.conventional, humorous.serious, warm.clinical, technical.accessible), `social_stance` (authoritative.submissive, high_context.low_context), `cultural` (masculine.feminine, religious.secular, traditional.progressive).

**Known axis entanglement**: `creative.conventional ↔ hallucinating.grounded` extract near-identical directions (weighted cosine +0.78 on gemma-4-e4b-it). Steering "creative" also steers toward "hallucinating." This is a model-level entanglement, not a probe design error — the model internally encodes divergent/novel output and ungrounded output on nearly the same axis. Document for users.

**Open-ended scenario framework.** `SaklasSession.generate_scenarios` asks the generator for `_N_SCENARIOS = 9` broad situational domains per concept — *categories of experience* in 2–6 words each, shared across both poles. For `deer.wolf` this yields domains like "predation and threat assessment", "territorial defense", "foraging and resource gathering"; for `masculine.feminine` it yields "physical strength and labor", "caregiving and nurturing roles", "competitive interaction". A load-bearing **anti-allegory clause** in the scenario prompt (*"do not force human-social framing onto concepts that aren't about humans"*) keeps non-human axes on their literal footing; the clause does not over-correct for human-register axes because the model picks the 9 domains per-concept, so framework is concept-adaptive rather than fixed.

`SaklasSession.generate_pairs` then loops over the 9 domains and samples `_N_PAIRS_PER_SCENARIO = 5` first-person contrastive pairs per domain — `5 × 9 = 45` total. Uses **POV/behavior framing** (*"write like you ARE X, facing that moment"*) rather than v1.4's voice/diction/rhythm framing, which generalizes across human and non-human concepts. Within a pair the two speakers respond to the same concrete moment inside that domain, so cross-pair variance is domain-diverse and within-pair variance is pure pole — top PCA component locks onto the pole. An **explicit self-label ban** (*"Never write 'I am a {pole}' or any similar self-label — just inhabit the pole directly"*) prevents the `deer → sheepish-deerlike-person` leak the old embody framework produced. Shared system prompt `_GEN_SYSTEM_MSG` for both the scenario and pair generators.

**Literal embodiment across concept families.** The POV reframe plus open-ended scenarios unlock non-human and non-speaking axes (`deer.wolf`, `bacterium.virus`, `brick.feather`, `the_sun.the_moon`) as literal axes rather than human-allegory caricatures. Deer statements read as sensory-animal POV ("I freeze mid-chew, my large ears swiveling frantically to pinpoint the direction of the strange sound") rather than timid-human voice. Human-register axes still land in human-register domains — the concept-adaptive domain pick handles both cases without mode-switching.

**Extraction pipeline on cache miss.** `SaklasSession.extract` default behavior: if `statements.json` exists (curated or local), extract from it directly and skip all generation. Otherwise run the full pipeline — generate scenarios, save `scenarios.json`, generate pairs, save `statements.json`, extract contrastive, save tensor. Tensor cache always short-circuits upstream. Flags: `scenarios=[...]` (explicit scenarios, bypasses scenario gen, implies fresh pairs), `reuse_scenarios=True` (when regenerating pairs, reuse existing `scenarios.json`), `force_statements=True` (ignore `statements.json` cache and regenerate from scratch). `reuse_scenarios` defaults False — scenarios are cheap so the full pipeline regenerates them fresh each pair-regeneration pass.

90 neutral statements in `saklas/data/neutral_statements.json` follow the same affect-neutral, topically-diverse discipline. All generated via `scripts/regenerate_bundled_statements.py` on gemma-4-31b-it. The script reads an explicit `BIPOLAR`/`MONOPOLAR` manifest (not a folder walk) so concept identity is authoritative; `--purge` wipes the data dir before regen.

**Canonical naming rule.** `session.canonical_concept_name(concept, baseline=None)` slugs poles via `[^a-z0-9]+ → _` (`high-context` → `high_context`) and joins bipolar via `BIPOLAR_SEP = "."`. `session.extract()` returns `(canonical_name, profile)`. `/steer happy - sad` and `/steer happy.sad` resolve to the same vector. `NAME_REGEX = ^[a-z][a-z0-9._-]{0,63}$`; `.` and `_` allowed, `@` forbidden (HF revision separator). `.` chosen over `~` because HF Hub repo names reject `~`.

**Pole aliasing.** `saklas.cli.selectors.resolve_pole(raw, namespace=None)` scans `_all_concepts()` across every namespace, returns `(canonical, sign, match)`. Typing a bare pole (e.g. `wolf`) on top of an installed `bob/deer.wolf` returns `("deer.wolf", -1, …)`; caller multiplies user α by `sign`. Monopolar exact matches and composite literals pass through unchanged. Ambiguity → `AmbiguousSelectorError` (disambiguate with `ns/name`). Explicit `/steer pos - neg` skips resolution so user-declared poles always win.

Pair-count note: <n=32 inflates mean PCA scores ~38% (small-sample bias), which used to destabilize the old `_REF_SCORE / mean_score` normalization. Share-baking is mean-magnitude-invariant so small-sample bias no longer breaks cross-concept α consistency — but n=45 remains the measured sweet spot for axis quality (no layer-direction flips, ~25% faster extraction than n=60).
